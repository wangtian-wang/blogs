性能优化 #

当我们在谈论性能优化的时候,我们的目的是啥 我们是否有一个明确的指标来判断性能优化前后的变化.记得之前在一家公司时,我们的 team leader 说对我们的 低代码解释器 做了优化.我就问了一句,你的优化指标是啥,然后被他理解为 我认为他们做的优化没有意义,于是乎,他告诉我说,软件的优化 过程 就是一次又一次推倒了再重来的过程... 我会想起自己之前曾经做过一些性能优化方面的工作,总结了以下内容.

找准问题,明确优化指标. #

个人觉得,要想知道我们应该怎样进行性能优化,首先需要理解网页的渲染过程,具体文章请参考: .当我们理解了这个之后,就会知道页面渲染的关键点在哪里.比如 资源请求的时间,资源请求的体积, 那些资源影响浏览器的渲染等.以下是我实践的总结.

  • 网络请求的优化
  • 浏览器渲染的优化

  • chrome performance panel

常见的性能优化指标 #

  • First Paint
    • 当渲染进程确认要渲染当前响应资源后,会先创建一个空白页面, 这个创建空白页面的时间点成为首次绘制时间.
  • First Conetent Paint -当页面绘制了第一个像素时(可以是text svg canvas 等),这个时间点 称为首次内容绘制时间.
  • Large Content Paint
    • 当首屏内容完全绘制完成时, 这个时间节点称为最大内容绘制时间,也称为首屏时间 . .
    • 标准为: LCP < 2.5S
  • First Input Delay
    • 用户第一次在页面上交互,到浏览器实际开始处理这个事件.
    • FIP < 100MS
  • Time To First Byte
    • 浏览器开始收到服务器响应数据的时间(后台处理时间 + 重定向时间),反应服务器响应速度的重要指标.
    • TTFB < 500ms, 否则 用户在打开网页时,就会感到明显的等待.
  • Cumulative Layout Shift
    • 视觉稳定性的测量指标, 以便提供良好的用户体验.
    • CLS 应该保持在 0.1s 或者更少
  • Time to Interactive
    • 用户可以与浏览器进行交互的时间
  • DOMContentLoaded
    • DOM 内容加载完毕
  • load
    • 页面内容全部加载完毕触发
  • 白屏时间
    • 从 FP 到 FCP 这段时间, 叫做白屏时间.比如页面背景页面发生变化.

针对网络请求的优化 #

  • 1: 减少 cookie 的大小; 静态内容可以不携带 cookie 或者将 cookie 种在另一个静态的域
  • 2: 分包. 首屏优化: 将首屏不需要的资源单独抽离出来,用到的时候再加载. 业务中引入第三方库: 业务代码与库的代码分离,浏览器默认会将已经请求的文件缓存下来,假设文件的名称未发生变化,不会重新发起请求,减少 http 请求次数与数据量(假设没有采取分包策略,每次需要请求该包时,当第三方包的体积大时,带来的额外的请求负担) 分包带来的问题是, 请求变多. 可以合并一些公共资源,使用雪碧图等,来减少因为分包而增多的请求
  • 3: 合理的使用缓存;
    • 指定缓存时间 cache-control
    • 提供验证机制 last-modify/ etag
    • 同时指定缓存时间或者验证机制,才能保证 缓存生效; 没有指定验证 -> 会在每次都没有更新的情况下重发相同内容; 没有指定缓存时间-> 每次使用资源时,都多余进行验证检查
  • 4: 使用 proconnect DNS-prefetch 提前解析 DNS , preload, prefecth 加快资源的请求速度
    • 减少 DNS 查询
    • 减少重定向
  • 5: 如果有条件,升级 http1.1 为 http2.0
  • 6: 使用 CDN
  • 7: 按需引入资源; 懒加载
  • 8: 前端压缩资源后,给到后端.

针对渲染的优化 #

  • 避免 JS 阻塞,如 大量的计算任务; 不合理的循环等
  • 避免重绘和回流带来的渲染性能
    • 1: 减少 dom 属性的操作, 尽量缓存类似于 offsetTop 等属性
    • 2: 尽量使用 transition; transform: willchange 等合成属性,避免对因改动一些 css 属性对其他元素造成影响
    • 3:css 的类名和 html DOM 嵌套的层级不要过深
    • 4: 关注继承属性
    • 5: 图片需要预先设置合适的尺寸,防止资源加载后 引发的回流
    html
    <img
  height="100px"
  srcset="
    /docs/public/assets/logo.png 100w,
    /docs/public/assets/logo.png 400w
  "
/>

日常开发的编码优化 #

  • 1: 合理的使用闭包
  • 2: dom 节点的引用要释放
  • 3: 定时器的清除
  • 4: 使用动画的时候,尽量使用 requestAnimationFrame()requestIdleCallback()16.6s 更新一次.js 假设 16.6s,页面的重排 重绘就会没时间执行,页面就会掉帧.
  • 5: 事件的防抖和节流
  • 6: 事件的解绑
  • 7: 作用域的控制 let len = arr.length
  • 8: 长列表的虚拟滚动和分页优化等.
  • 9: 大文件分片上传
  • 10: 懒加载
  • 11: 白屏优化
  • 12 : 服务端渲染
