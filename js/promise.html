# promise

# Promise 对象用于表示一个异步操作的最终完成（或失败）及其结果值。

# promise描述

promise能够把异步操作最终的成功返回值或者失败原因和相应的处理程序关联起来。使的异步方法可以向同步方法那样返回值(异步方法不会立即返回最终的值,而是返回一个promise,以便在未来某个时候把值交给使用者).

[注]:

promise并没有抛弃回调函数,只是改变了回调函数的调用位置 来自<<你不知道的javascript>>

# promise状态

  • 待定（pending）：初始状态，既没有被兑现，也没有被拒绝。
  • 已兑现（fulfilled）：意味着操作成功完成。
  • 已拒绝（rejected）：意味着操作失败。

# promise的方法

  1. 实例方法

    1. then
    2. catch
    3. finally

  2. 静态方法

    1. Resolve 直接返回状态为fullfilled的promise

    2. Reject 直接返回状态为rejected的promise

    3. All 接收一个数组,数组项为promise

      1. ​ 假设接收的为[]数组,立即返回一个已完成(already resolve)状态的promise;
      2. ​ 假设有一个promise执行失败,那立即返回失败的promise;
      3. ​ 假设全部执行完成,返回成功的promise 返回值将会按照参数内的 promise 顺序排列，而不是由调用 promise 的完成顺序决定。

    4. any 接收一个数组,数组项为promise

      1. ​ 假设接收的为[]数组,立即返回一个状态为rejected的promise;
      2. ​ 假设任意一个promise的状态为fullfilled,那么返回状态为fullfilled的promise
      3. ​ 否则,返回rejected状态的promise和一个AggregateError 错误实例

    5. race 接收一个数组,数组项为promise

      1. ​ 假设接收的为[]数组,立即返回一个状态为pending的promise;
      2. ​ 返回的promise的状态由第一个确定状态的promise数组项的状态决定;
      3. ​

    6. allSettled

      1. ​ 接收一个数组,数组项为promise
      2. ​ 一旦所指定的 promises 集合中每一个 promise 已经完成，无论是成功的达成或被拒绝，未决议的 [Promise]将被异步完成 (promises中的所有promise都会被执行) . 所返回的 promise 的处理器将传入一个数组作为输入，该数组包含原始 promises 集中每个 promise 的结果。

      allSettled 与 all的区别

      allSettled

      ​ 适合 多个彼此不依赖的异步任务成功完成时，或者总是想知道每个promise的结果时，通常使用它

      all

      ​ 更适合彼此相互依赖或者在其中任何一个reject时立即结束。

# promise在开发中的使用场景

# Promise.retry

promise并没有retry的静态方法,这个方法是利用promise的特性,实现 将一个函数执行N次,N次之前的任意次只要成功,直接返回成功的promise.否则,尝试n次后返回失败的promise.代码如下

// while 版本实现的
Promise.retry = function (fn, limit) {
  return new Promise(async (resolve, reject) => {
    while (limit--) {
      try {
        let result = await fn();
        resolve(result);
        break;
      } catch (error) {
        if (!limit) {
          // 尝试的次数用完后 立即变为失败
          reject(error);
        }
      }
    }
  });
};

// 使用async 实现的
Promise.retryAsync = function retryAsync(fn, limit) {
  if (typeof limit !== "number") limit = 3;
  return new Promise((resolve, reject) => {
    let index = 0;
    async function inner() {
      index++;
      try {
        let res = await fn();
        resolve(res);
      } catch (err) {
        if (index < limit) {
          inner();
        } else {
          reject(err);
        }
      }
    }
    inner();
  });
};
// 测试代码


function getNum() {
  return new Promise((resolve, reject) => {
    const num = Math.random() * 0.5;
    setTimeout(() => {
      if (num > 0.4) {
        resolve(`${num}ok`);
      } else {
        // console.log("failing" + num);
        reject("error");
      }
    }, 1000);
  });
}

# Promise.retryWithResult

该方法的功能是执行一个函数n次,假设n次之前的那次成功了,就返回n次之前尝试的所有结果,否则 尝试n次,返回所有的结果

Promise.retryWithResult = function retryWithResult(fn, times) {
  return new Promise((resolve, reject) => {
    let result = [],
      count = 0;
    async function run() {
      let index = count++;
      try {
        let res = await fn();
        result[index] = res;
        resolve(result);
      } catch (err) {
        result[index] = err;
        if (count < times) {
          run();
        } else {
          reject(result);
        }
      }
    }
    run();
  });
};

Promise.retryWithAllResult = (fn, time) => {
  return new Promise((resolve, reject) => {
    let index = 0, // 当前执行到第几次
      result = [];
    while (index < time) {
      let i = index; // 保存当前次执行结果的index
      index++;
      fn().then(
        (res) => {
          if (i === time - 1) {
            resolve(result); // // 执行resolve后 后面的代码  result[i] = res;还能执行
          }
          result[i] = res;
        },
        (error) => {
          if (i === time - 1) {
            reject(result);
          }
          result[i] = error;
        }
      );
    }
  });
};

该函数的功能是让一个异步函数 ,间隔n秒 ,执行一次.

    }, 1000);

// 正式函数
function repeat(fn, times, delay) {
  return async function repeat(...args) {
    for (let i = 0; i < times; i++) {
      let res = await new Promise((resolve) => {
        setTimeout(() => {
          resolve(fn.call(this, ...args, i));
        }, 1000);
      });
      console.log(res);
    }
  };
}



async function repeatWithForLoop(cb, times, delay) {
  for (let i = 0; i < times; i++) {
    let res = await square(cb, i, delay);
    console.log(res);
  }
}

# asyncControl

控制异步任务的并发数量(同时运行n个异步任务), 返回每个任务的结果.

function requestControl(tasks, pool) {
  if (typeof pool !== "number") pool = 2;
  if (!Array.isArray(tasks)) return;
  let index = 0,
    result = [];
  let requestPool = new Array(pool).fill(null);
  //  map的作用 : 只是开启了2个任务池, 前2个任务会被抛进任务池,
  requestPool = requestPool.map(() => {
    return new Promise((resolve, reject) => {
      function runner() {
        if (index >= tasks.length) {
          resolve(result);
        }
        let curIndex = index;
        let curTask = tasks[index++];
        if (!curTask instanceof Promise) curTask = Promise.resolve(curTask);
        curTask()
          .then((res) => {
            result[curIndex] = res;
            runner();
          })
          .catch((error) => {
            reject(error);
          });
      }
      runner();
    });
  });
  // promise.all执行的时候, 会在前2个任务的.then里面 调用run()函数 用下一个函数来 代替最初的taskArray中的第一个位置的函数 循环执行
  return Promise.all(requestPool).then((res) => result);
}

# 如何判断一个值是否是promise

  • 使用instanceof

    instanceof 运算符用于检测构造函数的 prototype 属性是否出现在某个实例对象的原型链上。 但是我们常用的Promise是 ES6对Promise/A+ 的一种实现，开源社区中的一些库是自己实现的，比如 bluebird (opens new window)。所以我们使用 ES6 原生 Promise 去校验 bluebird 的话肯定是校验不通过的。

  • 使用toString()方法判断

    function isPromise(p) {
  return p && Object.prototype.toString.call(p) === "[object Promise]";
}
